📚 Tổng hợp Kiểu Dữ Liệu C++

Hướng dẫn chi tiết về kích thước và chức năng các kiểu biến

Tổng Quan Kiểu Dữ Liệu Cơ Bản

Kiểu Dữ Liệu Kích Thước Phạm Vi Giá Trị Chức Năng
bool 1 byte true hoặc false Lưu giá trị logic đúng/sai
char 1 byte -128 đến 127 Lưu ký tự ASCII
unsigned char 1 byte 0 đến 255 Ký tự không dấu
short 2 bytes -32,768 đến 32,767 Số nguyên ngắn
int 4 bytes -2,147,483,648 đến 2,147,483,647 Số nguyên tiêu chuẩn
long 4/8 bytes Tùy hệ thống (32/64 bit) Số nguyên dài
long long 8 bytes -9,223,372,036,854,775,808 đến 9,223,372,036,854,775,807 Số nguyên rất dài
float 4 bytes ±3.4E-38 đến ±3.4E+38 (7 chữ số thập phân) Số thực độ chính xác đơn
double 8 bytes ±1.7E-308 đến ±1.7E+308 (15 chữ số thập phân) Số thực độ chính xác kép
long double 8/16 bytes Tùy hệ thống (19+ chữ số thập phân) Số thực độ chính xác mở rộng

💡 Lưu Ý

  • Kích thước có thể khác nhau tùy compiler và hệ thống (32-bit vs 64-bit)
  • Dùng sizeof() để kiểm tra kích thước chính xác
  • Chọn kiểu dữ liệu phù hợp để tối ưu bộ nhớ

Kiểu Số Nguyên Chi Tiết

Kiểu Signed Unsigned Khi Nào Dùng
char -128 đến 127 0 đến 255 Ký tự, số nhỏ, byte
short -32,768 đến 32,767 0 đến 65,535 Tiết kiệm bộ nhớ, giá trị nhỏ
int ~-2.1 tỷ đến ~2.1 tỷ 0 đến ~4.2 tỷ Mặc định cho số nguyên

// Ví dụ khai báo char age = 25; unsigned char byte_data = 255; short year = 2025; int population = 1000000; unsigned int positive_only = 4000000000; long long big_number = 9223372036854775807LL;

⚠️ Lưu Ý Overflow

Khi giá trị vượt quá phạm vi, sẽ xảy ra tràn số (overflow):

  • Signed overflow: Kết quả không xác định (undefined behavior)
  • Unsigned overflow: Quay vòng (wrap around): 255 + 1 = 0

Kiểu Số Thực (Floating Point)

Kiểu Kích Thước Độ Chính Xác Phạm Vi Khi Nào Dùng
float 4 bytes ~7 chữ số ±3.4E±38 Đồ họa, game, tiết kiệm bộ nhớ
double 8 bytes ~15 chữ số ±1.7E±308 Mặc định, tính toán khoa học
long double 8-16 bytes ~19+ chữ số Tùy compiler Tính toán siêu chính xác
// Ví dụ sử dụng float pi_f = 3.14159f; // Thêm 'f' cho float double pi_d = 3.141592653589793; // Mặc định là double long double pi_ld = 3.141592653589793238L; // Thêm 'L' // Biểu diễn khoa học float small = 1.23e-10f; // 1.23 × 10^-10 double large = 6.022e23; // 6.022 × 10^23 (số Avogadro)

⚠️ Vấn Đề Độ Chính Xác

  • Số thực không lưu chính xác 100%: 0.1 + 0.2 ≠ 0.3
  • Không nên so sánh trực tiếp: if (a == b)
  • Dùng epsilon: if (abs(a - b) < 1e-9)

💡 Khi Nào Dùng float vs double?

  • float: Mảng lớn (hình ảnh, âm thanh), game, GPU
  • double: Tính toán tài chính, khoa học, mặc định
  • long double: Thiên văn học, mật mã học

Kiểu Kích Thước Mô Tả Ví Dụ

💡 Escape Sequences Thường Dùng

  • \n - Xuống dòng (newline)
  • \t - Tab ngang
  • \\ - Dấu backslash
  • \' - Dấu nháy đơn
  • \" - Dấu nháy kép
  • \0 - Null character (kết thúc string)

Con Trỏ (Pointer) và Tham Chiếu (Reference)

Loại Cú Pháp Kích Thước Đặc Điểm
Pointer int* ptr; 4 bytes (32-bit)
8 bytes (64-bit)		 Lưu địa chỉ bộ nhớ, có thể NULL, thay đổi được
Reference int& ref; 0 bytes* Alias của biến, phải khởi tạo, không thay đổi
Pointer to Pointer int** pptr; 4/8 bytes Con trỏ trỏ đến con trỏ
Array Pointer int* arr; 4/8 bytes Trỏ đến phần tử đầu mảng
Function Pointer int (*fptr)(int); 4/8 bytes Trỏ đến hàm
// Con trỏ (Pointer) int x = 10; int* ptr = &x; // ptr lưu địa chỉ của x cout << *ptr; // 10 (dereference để lấy giá trị) *ptr = 20; // Thay đổi x thành 20 ptr = nullptr; // Gán NULL (C++11) // Tham chiếu (Reference) int y = 30; int& ref = y; // ref là alias của y ref = 40; // Thay đổi y thành 40 // int& ref2; // LỖI: phải khởi tạo ngay // Pass by reference trong hàm void swap(int& a, int& b) { int temp = a; a = b; b = temp; }

⚠️ Lỗi Thường Gặp

  • Dangling pointer: Trỏ đến vùng nhớ đã bị giải phóng
  • Memory leak: Quên delete sau new
  • Null pointer dereference: Truy cập *ptr khi ptr = NULL
  • Wild pointer: Pointer chưa khởi tạo

💡 Khi Nào Dùng Pointer vs Reference?

  • Reference: Tham số hàm, không cần kiểm tra NULL, đơn giản
  • Pointer: Cần NULL, cấp phát động, thay đổi địa chỉ

Kiểu Dữ Liệu Phức Hợp

1. Array (Mảng)

// Mảng tĩnh int arr[5] = {1, 2, 3, 4, 5}; // 5 * 4 = 20 bytes char str[50]; // 50 bytes double matrix[3][4]; // 3 * 4 * 8 = 96 bytes // Mảng động (C++) int* dynamic_arr = new int[n]; // n * 4 bytes delete[] dynamic_arr; // Phải giải phóng!

2. Struct (Cấu trúc)

struct SinhVien { char hoTen[50]; // 50 bytes int mssv; // 4 bytes float diemTB; // 4 bytes // Padding để căn chỉnh: thường 2 bytes }; // Tổng: ~60 bytes (có thể khác do padding) SinhVien sv1; sizeof(SinhVien); // Kiểm tra kích thước thực tế

3. Union

union Data { int i; // 4 bytes float f; // 4 bytes char str[20]; // 20 bytes }; // Kích thước = phần tử lớn nhất = 20 bytes Data d; d.i = 10; // Dùng như int d.f = 3.14; // Ghi đè lên i (chia sẻ vùng nhớ)

4. Enum (Liệt kê)

// C-style enum enum Color { RED, GREEN, BLUE }; // 4 bytes (như int) Color c = RED; // C++11 enum class (khuyên dùng) enum class Status : char { OK, ERROR, PENDING }; // 1 byte Status s = Status::OK;

5. Class

class Student { private: string name; // ~32 bytes (tùy implementation) int age; // 4 bytes float gpa; // 4 bytes public: void display(); // Hàm không chiếm bộ nhớ trong object }; // Kích thước tùy thuộc vào member variables + padding

💡 Data Alignment & Padding

Compiler tự động thêm padding để tối ưu hiệu suất:

  • Mỗi kiểu dữ liệu cần căn chỉnh theo bội số kích thước của nó
  • double (8 bytes) → địa chỉ chia hết cho 8
  • int (4 bytes) → địa chỉ chia hết cho 4
  • Sắp xếp member từ lớn đến nhỏ để giảm padding

⚠️ Tối Ưu Struct

// ❌ Không tối ưu (72 bytes) struct Bad { char c; // 1 byte double d; // 8 bytes (cần 7 bytes padding trước) int i; // 4 bytes }; // ✅ Tối ưu (16 bytes) struct Good { double d; // 8 bytes int i; // 4 bytes char c; // 1 byte (3 bytes padding sau) };

6. Vector & Container (C++ STL)

#include <vector> #include <map> #include <set> vector<int> vec; // ~24 bytes overhead + data vec.push_back(10); vec.size(); // Số phần tử vec.capacity(); // Dung lượng đã cấp phát map<string, int> m; // Overhead lớn hơn set<int> s; // Dữ liệu được sắp xếp
